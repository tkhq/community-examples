## Overview

This guide shows how to use Turnkey embedded wallets to interact with Zama confidential tokens on Sepolia.

[Zama's Confidential Token standard](https://docs.zama.org/protocol) wraps ERC-20 tokens into privacy-preserving counterparts where balances and transfer amounts are encrypted on-chain. Users can:

- **Shield** — lock ERC-20 tokens into a confidential wrapper
- **Decrypt balance** — read their encrypted balance with a wallet-signed session key
- **Transfer confidentially** — send tokens without exposing the amount on-chain
- **Unshield** — convert confidential tokens back to standard ERC-20 (two-phase)

Turnkey handles key management and transaction signing. The Zama SDK handles all encryption and on-chain coordination. The integration point is Zama's `GenericSigner` interface, implemented here as a thin adapter over Turnkey-backed viem clients.

## Integration steps

### 1. Implement `TurnkeyViemSigner`

The Zama SDK requires a `GenericSigner` — an object that wraps a signing account and exposes standardized methods for reading/writing contracts and signing EIP-712 typed data.

`ViemSigner` (from `@zama-fhe/sdk/viem`) expects an EIP-1193 `ethereum` provider for wallet lifecycle events, which Turnkey does not expose. The solution is a thin adapter that implements `GenericSigner` directly on top of viem clients backed by a Turnkey account.

Two critical implementation details:
- Strip `EIP712Domain` from `typedData.types` before calling `walletClient.signTypedData` — viem injects it automatically from the `domain` field and will throw on duplicates.
- `primaryType` is typed as optional in `GenericSigner`'s `EIP712TypedData` — derive it from the remaining keys as a fallback when not provided.

```tsx
// src/lib/turnkey-viem-signer.ts
import type {
  Account,
  Abi,
  ContractFunctionArgs,
  ContractFunctionName,
  ContractFunctionReturnType,
  PublicClient,
  WalletClient,
  Address,
  Hex,
} from "viem";
// Used for its type only: satisfies walletClient.writeContract's parameter shape.
import type { writeContract } from "viem/actions";
import type {
  EIP712TypedData,
  GenericSigner,
  ReadContractConfig,
  SignerLifecycleCallbacks,
  TransactionReceipt,
  WriteContractConfig,
} from "@zama-fhe/sdk";

export interface TurnkeyViemSignerConfig {
  walletClient?: WalletClient;
  publicClient: PublicClient;
}

export class TurnkeyViemSigner implements GenericSigner {
  readonly #walletClient?: WalletClient;
  readonly #publicClient: PublicClient;

  constructor(config: TurnkeyViemSignerConfig) {
    this.#walletClient = config.walletClient;
    this.#publicClient = config.publicClient;
  }

  #requireWalletClient(): WalletClient {
    if (!this.#walletClient) {
      throw new TypeError("No walletClient configured — read-only mode");
    }
    return this.#walletClient;
  }

  #requireWalletAndAccount(): { walletClient: WalletClient; account: Account } {
    const walletClient = this.#requireWalletClient();
    if (!walletClient.account) {
      throw new TypeError("WalletClient has no account");
    }
    return { walletClient, account: walletClient.account };
  }

  async getChainId(): Promise<number> {
    return this.#publicClient.getChainId();
  }

  async getAddress(): Promise<Address> {
    return this.#requireWalletAndAccount().account.address;
  }

  async signTypedData(typedData: EIP712TypedData): Promise<Hex> {
    const { walletClient, account } = this.#requireWalletAndAccount();

    // viem re-injects EIP712Domain from the `domain` field — passing it
    // inside `types` causes a duplicate-type error.
    const { EIP712Domain: _, ...sigTypes } = typedData.types;

    // primaryType is optional in EIP712TypedData — fall back to the first key
    // in sigTypes when not provided.
    const primaryType =
      typedData.primaryType ?? (Object.keys(sigTypes)[0] as string);

    // Safely coerce only the fields that are known to arrive as numbers from
    // the Zama SDK worker but must be bigint for viem's uint256 encoding.
    const message = { ...typedData.message } as Record<string, unknown>;
    if ("startTimestamp" in message && typeof message.startTimestamp === "number") {
      message.startTimestamp = BigInt(message.startTimestamp);
    }
    if ("durationDays" in message && typeof message.durationDays === "number") {
      message.durationDays = BigInt(message.durationDays);
    }

    return walletClient.signTypedData({
      account,
      primaryType,
      types: sigTypes,
      domain: typedData.domain,
      message,
    } as Parameters<typeof walletClient.signTypedData>[0]);
  }

  async writeContract<
    const TAbi extends Abi | readonly unknown[],
    TFunctionName extends ContractFunctionName<TAbi, "nonpayable" | "payable">,
    const TArgs extends ContractFunctionArgs<
      TAbi,
      "nonpayable" | "payable",
      TFunctionName
    >,
  >(config: WriteContractConfig<TAbi, TFunctionName, TArgs>): Promise<Hex> {
    const { walletClient, account } = this.#requireWalletAndAccount();
    return walletClient.writeContract({
      chain: walletClient.chain,
      account,
      ...config,
    } as Parameters<typeof writeContract>[1]);
  }

  async readContract<
    const TAbi extends Abi | readonly unknown[],
    TFunctionName extends ContractFunctionName<TAbi, "pure" | "view">,
    const TArgs extends ContractFunctionArgs<TAbi, "pure" | "view", TFunctionName>,
  >(
    config: ReadContractConfig<TAbi, TFunctionName, TArgs>,
  ): Promise<
    ContractFunctionReturnType<TAbi, "pure" | "view", TFunctionName, TArgs>
  > {
    return this.#publicClient.readContract(config);
  }

  async waitForTransactionReceipt(hash: Hex): Promise<TransactionReceipt> {
    return this.#publicClient.waitForTransactionReceipt({ hash });
  }

  async getBlockTimestamp(): Promise<bigint> {
    const block = await this.#publicClient.getBlock();
    return block.timestamp;
  }

  subscribe(_callbacks: SignerLifecycleCallbacks): () => void {
    // Turnkey has no EIP-1193 provider — wallet lifecycle events are not supported.
    return () => {};
  }
}
```

### 2. Set up providers

#### Chain configuration

```tsx
// src/lib/chain-config.ts
import { SepoliaConfig } from "@zama-fhe/sdk";
```

The snippets below are focused extracts. In a real app, keep `signer` and `relayer` in component state and initialize them inside `useEffect` after the Turnkey session is ready.

#### Step 1 — Wrap the app

Import the Wallet Kit stylesheet in your root layout (a Server Component), not in the client providers file:

```tsx
// src/app/layout.tsx
import "@turnkey/react-wallet-kit/styles.css";
```

Set up top-level providers. `TurnkeyProvider` handles wallet authentication; `QueryClientProvider` is required by the Zama React hooks:

```tsx
// src/components/providers.tsx
"use client";
import { type ReactNode } from "react";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { TurnkeyProvider } from "@turnkey/react-wallet-kit";

const queryClient = new QueryClient();

export function Providers({ children }: { children: ReactNode }) {
  return (
    <QueryClientProvider client={queryClient}>
      <TurnkeyProvider
        config={{
          organizationId: process.env.NEXT_PUBLIC_TURNKEY_ORG_ID!,
          authProxyConfigId: process.env.NEXT_PUBLIC_TURNKEY_AUTH_PROXY_CONFIG_ID!,
        }}
      >
        {children}
      </TurnkeyProvider>
    </QueryClientProvider>
  );
}
```

#### Step 2 — Create the Turnkey-backed signer

Inside a child component wrapped by `TurnkeyProvider`, retrieve the wallet account and build a `TurnkeyViemSigner`. Run this inside a `useEffect` that triggers once `httpClient` and `wallets` are available:

```tsx
// Inside ZamaLayer — a child of TurnkeyProvider
import { useTurnkey, ClientState } from "@turnkey/react-wallet-kit";
import { createAccount } from "@turnkey/viem";
import { createWalletClient, createPublicClient, http, isAddress } from "viem";
import { TurnkeyViemSigner } from "@/lib/turnkey-viem-signer";
import { viemChain } from "@/lib/chain-config";

const { httpClient, wallets, fetchWalletAccounts, clientState } = useTurnkey();

async function createTurnkeySigner() {
  if (clientState !== ClientState.Ready || !httpClient || wallets.length === 0) return null;

  const accounts = await fetchWalletAccounts({ wallet: wallets[0] });
  const ethAccount = accounts.find((account) => isAddress(account.address));

  if (!ethAccount) {
    throw new Error("No Ethereum account found in the selected Turnkey wallet");
  }

  const account = await createAccount({
    client: httpClient,
    organizationId: ethAccount.organizationId,
    signWith: ethAccount.address,
    ethereumAddress: ethAccount.address,
  });

  const walletClient = createWalletClient({
    account,
    chain: viemChain,
    transport: http(process.env.NEXT_PUBLIC_RPC_URL),
  });

  const publicClient = createPublicClient({
    chain: viemChain,
    transport: http(process.env.NEXT_PUBLIC_RPC_URL),
  });

  return new TurnkeyViemSigner({ walletClient, publicClient });
}
```

#### Step 3 — Initialize Zama

Initialize the relayer inside a `useEffect` (requires browser environment for `window.location.origin`) and render `ZamaProvider` once `signer` and `relayer` are ready. The relayer URL is proxied through `/api/relayer` to avoid CORS and keep the API key server-side. Two storage instances keep the encrypted keypair and session signatures isolated:

```tsx
// src/components/providers.tsx (continued)
import {
  ZamaProvider, RelayerWeb, IndexedDBStorage, indexedDBStorage,
  ZamaSDKEvents, savePendingUnshield,
} from "@zama-fhe/react-sdk";
import type { Address } from "viem";
import { zamaConfig } from "@/lib/chain-config";

const sessionDBStorage = new IndexedDBStorage("SessionStore");

// Exported so UnshieldPanel can tell onEvent which token is being unshielded.
let activeUnshieldToken: Address | null = null;
export function setActiveUnshieldToken(addr: Address | null) {
  activeUnshieldToken = addr;
}

// Inside useEffect([]):
setRelayer(new RelayerWeb({
  getChainId: async () => zamaConfig.chainId,
  transports: {
    [zamaConfig.chainId]: {
      ...zamaConfig,
      relayerUrl: `${window.location.origin}/api/relayer`,
      network: process.env.NEXT_PUBLIC_RPC_URL,
    },
  },
}));

// Render once both relayer and signer are ready:
return (
  <ZamaProvider
    relayer={relayer}
    signer={signer}
    storage={indexedDBStorage}
    sessionStorage={sessionDBStorage}
    onEvent={(event) => {
      if (event.type === ZamaSDKEvents.UnshieldPhase1Submitted && activeUnshieldToken) {
        savePendingUnshield(indexedDBStorage, activeUnshieldToken, event.txHash).catch(console.error);
        activeUnshieldToken = null;
      }
    }}
  >
    {children}
  </ZamaProvider>
);
```

### 3. Add the relayer proxy API route

The Zama relayer may restrict direct browser access (CORS). The example below uses **Next.js App Router** (Next.js 15+). If you use another framework, implement the same same-origin proxy pattern in your backend.

```tsx
// src/app/api/relayer/[...path]/route.ts
import { NextRequest } from "next/server";

export const dynamic = "force-dynamic";

const RELAYER_BASE = "https://relayer.testnet.zama.org/v2";

// params is a Promise in Next.js 15+
type RouteContext = { params: Promise<{ path: string[] }> };

async function proxy(request: NextRequest, path: string[]): Promise<Response> {
  const target = new URL(`${RELAYER_BASE}/${path.join("/")}`);
  request.nextUrl.searchParams.forEach((value, key) => {
    target.searchParams.set(key, value);
  });

  const headers: Record<string, string> = {
    "Content-Type": request.headers.get("Content-Type") ?? "application/json",
  };

  const body =
    request.method !== "GET" && request.method !== "HEAD"
      ? await request.text()
      : undefined;

  let upstream: globalThis.Response;
  try {
    upstream = await fetch(target.toString(), {
      method: request.method,
      headers,
      body,
    });
  } catch (e: unknown) {
    const message = e instanceof Error ? e.message : "Upstream unreachable";
    return Response.json({ error: message }, { status: 502 });
  }

  return new Response(await upstream.text(), {
    status: upstream.status,
    headers: {
      "Content-Type":
        upstream.headers.get("Content-Type") ?? "application/json",
    },
  });
}

export async function GET(request: NextRequest, context: RouteContext) {
  return proxy(request, (await context.params).path);
}
export async function POST(request: NextRequest, context: RouteContext) {
  return proxy(request, (await context.params).path);
}
export async function PUT(request: NextRequest, context: RouteContext) {
  return proxy(request, (await context.params).path);
}
export async function DELETE(request: NextRequest, context: RouteContext) {
  return proxy(request, (await context.params).path);
}
```

### 4. Use the SDK hooks

With providers in place, the Zama React hooks work out of the box. Pass the **wrapper address** (confidential token) as `tokenAddress`.

#### Discover available token pairs

Use `useListPairs` to fetch the on-chain wrapper registry. This is the recommended way to build a token selector:

```tsx
import { useListPairs } from "@zama-fhe/react-sdk";
import type { Address } from "viem";
import type { TokenWrapperPairWithMetadata } from "@zama-fhe/react-sdk";

function TokenSelector({ onSelect }: { onSelect: (address: Address) => void }) {
  const { data, isPending } = useListPairs({ metadata: true });
  const pairs = (data?.items ?? []).filter(
    (p): p is TokenWrapperPairWithMetadata => p.isValid && "underlying" in p,
  );

  if (isPending) return <span>Loading tokens…</span>;

  return (
    <select onChange={(e) => onSelect(e.target.value as Address)}>
      {pairs.map((p) => (
        <option key={p.confidentialTokenAddress} value={p.confidentialTokenAddress}>
          {p.metadata?.symbol ?? p.confidentialTokenAddress}
        </option>
      ))}
    </select>
  );
}
```

#### Shield (ERC-20 → confidential)

Some RPC nodes simulate transactions against the committed chain state. When `approve` and `wrap` land in the same block, the wrap may revert because the node sees no approved allowance yet. The fix is to perform the approval yourself with a confirmed receipt, then pass `approvalStrategy: "skip"` so the SDK skips its internal approval step.

```tsx
import { useState } from "react";
import { useShield, allowanceContract, approveContract, useZamaSDK } from "@zama-fhe/react-sdk";
import type { Address, Hex } from "viem";

function ShieldPanel({
  wrapperAddress,
  underlyingAddress,
  ownerAddress,
}: {
  wrapperAddress: Address;
  underlyingAddress: Address;
  ownerAddress: Address;
}) {
  const { signer } = useZamaSDK();
  const shield = useShield({ tokenAddress: wrapperAddress });
  const [shieldError, setShieldError] = useState<string | null>(null);

  // Perform ERC-20 approval with a confirmed receipt before shielding.
  async function preApprove(amount: bigint): Promise<void> {
    const allowance = (await signer.readContract(
      allowanceContract(underlyingAddress, ownerAddress, wrapperAddress),
    )) as bigint;

    if (allowance >= amount) return;

    // Reset non-zero allowance first — required by USDT-like tokens.
    if (allowance > 0n) {
      const resetHash = await signer.writeContract(
        approveContract(underlyingAddress, wrapperAddress, 0n),
      );
      await signer.waitForTransactionReceipt(resetHash as Hex);
    }

    const approveHash = await signer.writeContract(
      approveContract(underlyingAddress, wrapperAddress, amount),
    );
    await signer.waitForTransactionReceipt(approveHash as Hex);
  }

  async function handleShield(amount: bigint) {
    setShieldError(null);
    try {
      await preApprove(amount);
    } catch (e) {
      setShieldError(e instanceof Error ? e.message : "Approval failed");
      return;
    }
    shield.mutate({ amount, approvalStrategy: "skip" });
  }

  return (
    <>
      {shieldError && <p style={{ color: "red" }}>{shieldError}</p>}
      <button onClick={() => handleShield(100_000_000n)} disabled={shield.isPending}>
        {shield.isPending ? "Shielding…" : "Shield 100 USDC"}
      </button>
    </>
  );
}
```

#### Decrypt confidential balance

Decrypting a confidential balance triggers an EIP-712 signature request: the user's wallet signs an authorization that allows the Zama relayer to perform the decryption on their behalf. The session key is cached after the first signature and reused until it expires (see [Session key re-authorization](#session-key-re-authorization) below).

```tsx
import { useConfidentialBalance } from "@zama-fhe/react-sdk";
import { formatUnits, type Address } from "viem";

function ConfidentialBalance({
  wrapperAddress,
  decimals,
  symbol,
}: {
  wrapperAddress: Address;
  decimals: number;
  symbol: string;
}) {
  const { data: balance, isLoading } = useConfidentialBalance({
    tokenAddress: wrapperAddress,
  });

  if (isLoading) return <span>Decrypting…</span>;
  return (
    <span>
      {balance !== undefined ? formatUnits(balance, decimals) : "—"} {symbol}
    </span>
  );
}
```

#### Session key re-authorization

Balance decryption uses a session key signed by the user's wallet. Sessions expire after a configurable period. Use `useIsAllowed` to check validity and `useAllow` to trigger re-authorization — typically shown as a gate before the balance component:

```tsx
import { useIsAllowed, useAllow } from "@zama-fhe/react-sdk";
import type { Address } from "viem";

function AuthGuard({
  wrapperAddress,
  children,
}: {
  wrapperAddress: Address;
  children: React.ReactNode;
}) {
  const { data: isAllowed } = useIsAllowed({ contractAddresses: [wrapperAddress] });
  const allow = useAllow();

  if (!isAllowed) {
    return (
      <button
        onClick={() => allow.mutate({ contractAddresses: [wrapperAddress] })}
        disabled={allow.isPending}
      >
        {allow.isPending ? "Authorizing…" : "Authorize balance decrypt"}
      </button>
    );
  }

  return <>{children}</>;
}
```

Wrap `<ConfidentialBalance>` with `<AuthGuard>` to automatically prompt re-authorization when the session expires.

#### Confidential transfer

```tsx
import { useState } from "react";
import { useConfidentialTransfer } from "@zama-fhe/react-sdk";
import { isAddress, type Address } from "viem";

function TransferPanel({ wrapperAddress }: { wrapperAddress: Address }) {
  const transfer = useConfidentialTransfer({ tokenAddress: wrapperAddress });
  const [recipient, setRecipient] = useState("");

  function handleTransfer(amount: bigint) {
    if (!isAddress(recipient)) {
      throw new Error("Enter a valid Ethereum recipient address");
    }
    transfer.mutate({ to: recipient, amount });
  }

  return (
    <>
      <input
        value={recipient}
        onChange={(event) => setRecipient(event.target.value)}
        placeholder="0x..."
      />
      {transfer.error && <p style={{ color: "red" }}>{transfer.error.message}</p>}
      <button
        onClick={() => handleTransfer(50_000_000n)}
        disabled={transfer.isPending || !isAddress(recipient)}
      >
        {transfer.isPending ? "Sending…" : "Transfer 50 USDC confidentially"}
      </button>
    </>
  );
}
```

#### Unshield (confidential → ERC-20)

Unshield is a **two-phase** operation and is not immediate. Phase 1 submits the on-chain unwrap request; phase 2 waits for the KMS to decrypt the ciphertext and finalize the withdrawal — tokens remain locked until phase 2 completes. If the user closes the app between phases, the pending unwrap hash is persisted and can be resumed.

```tsx
import {
  useUnshield,
  useResumeUnshield,
  clearPendingUnshield,
  indexedDBStorage,
} from "@zama-fhe/react-sdk";
import type { Hex, Address } from "viem";
import { setActiveUnshieldToken } from "@/components/providers";

function UnshieldPanel({
  wrapperAddress,
  pendingTxHash,
  onSuccess,
}: {
  wrapperAddress: Address;
  pendingTxHash: Hex | null;
  onSuccess: () => void;
}) {
  const unshield = useUnshield({
    tokenAddress: wrapperAddress,
    wrapperAddress,
  });
  const resumeUnshield = useResumeUnshield({
    tokenAddress: wrapperAddress,
    wrapperAddress,
  });

  function handleUnshield(amount: bigint) {
    setActiveUnshieldToken(wrapperAddress);
    unshield.mutate(
      { amount },
      {
        onSuccess: () => {
          clearPendingUnshield(indexedDBStorage, wrapperAddress).catch(
            console.error,
          );
          onSuccess();
        },
        onError: () => setActiveUnshieldToken(null),
      },
    );
  }

  function handleResume(txHash: Hex) {
    resumeUnshield.mutate(
      { unwrapTxHash: txHash },
      {
        onSuccess: () => {
          clearPendingUnshield(indexedDBStorage, wrapperAddress).catch(
            console.error,
          );
          onSuccess();
        },
      },
    );
  }

  return (
    <>
      {(unshield.error ?? resumeUnshield.error) && (
        <p style={{ color: "red" }}>
          {(unshield.error ?? resumeUnshield.error)?.message}
        </p>
      )}
      {pendingTxHash && (
        <button
          onClick={() => handleResume(pendingTxHash)}
          disabled={resumeUnshield.isPending}
        >
          {resumeUnshield.isPending
            ? "Resuming…"
            : "Resume unshield (phase 2)"}
        </button>
      )}
      <button
        onClick={() => handleUnshield(50_000_000n)}
        disabled={unshield.isPending}
      >
        {unshield.isPending ? "Unshielding…" : "Unshield 50 USDC"}
      </button>
    </>
  );
}
```

- Authenticate users with `@turnkey/react-wallet-kit`
- Create a Turnkey-backed viem account with `@turnkey/viem`
- Adapt that account to the Zama SDK through `GenericSigner`

- Shield ERC-20 tokens into confidential wrappers
- Decrypt confidential balances with an EIP-712 session authorization
- Send confidential transfers
- Unshield tokens through the two-phase withdrawal flow
